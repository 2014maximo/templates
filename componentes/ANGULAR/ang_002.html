PRUEBAS UNITARIAS EN ANGULAR

Aclarando el término de pruebas unitarias, que se refiere a realizarle pruebas de correcto funcionamiento a unidades de código, pequeñas partes, muy concretamente me refiero a las funciones, variables, peticiones, métodos, a clases, la carga de componentes en la vista, o sea simplificando, realizarle pruebas de que si sirve, a todo lo que construímos como desarrolladores frontend en Angular.

Se vuelve imprescindible el tema de pruebas unitarias para garantizar los desarrollos que entregamos a clientes o empresas, que de hecho facilita mucho una parte del camino de la calidad de los desarrollos. O sea que es infaltable este requisito para un desarrollador de Angular.

Más adelante se muestran ejemplos de pruebas realizadas a código, y, a continuación, se describen los elementos que hacen parte del proceso de pruebas unitarias con Karma Jasmine:

1

¿Qué se necesita?

La parte realmente importante en pruebas unitarias son las evaluaciones a los métodos, teniendo en cuenta que se necesita simular todos los requisitos y dependencias que necesita ese método, función, clase, etc, para entregar el resultado del proceso.
Anterior a esto debemos contar con estos elementos:

Las librerías que hacen las pruebas

npm i karma-jasmine

"karma-jasmine"

npm i karma-jasmine

Los proyectos de Angular por defecto ya traen la dependencia de Karma Jasmine para las pruebas unitarias. De requerirse se puede actualizar la versión de Karma que esta actualmenten la versión 4 con:
npm i karma-jasmine

Archivos ".spec"

Los archivos .spec son la mesa de trabajo para realizar las pruebas, y sobre ellos se realizan los diagnósticos de estadísticas generales de pruebas de todo el proyecto. Por estandar y fácil localización se crean en la misma ruta del componente que se va a probar.

2

¿Qué contienen las pruebas?

En los archivos .spec debe ir la extructura de funciones que empieza por la función "describe", que contiene a todas las demás. De esta manera:

Función "describe('', ()=>{})"

La función "describe()", es la que contiene todo el area de pruebas, recibe 2 parámetros que serían la descripción que es una cadena de texto y recibe otra función que es la que contiene todo el área de pruebas.

Función "beforeEach(()=>{})"

Esta función ejecuta todo lo que tiene dentro de si, antes de cada una de las pruebas.

Función "beforeAll(()=>{})"

Esta función ejecuta todo lo que tiene dentro de si, antes de todas las pruebas.

Función "afteEach(()=>{})"

Esta función ejecuta todo lo que tiene dentro de si, despues de cada una de las pruebas.

Función "afteAll(() => {})"

Esta función ejecuta todo lo que tiene dentro de si, despues de todas las pruebas.

Función "it('' () =>{})"

Dentro de esta función en específico, se construye la evaluación.

Función "expect()."

Esta función es la encargada de ejecutar una comparación del resultado esperado o fallido.
